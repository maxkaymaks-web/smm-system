# Скилл: Проверка и редактура поста

## Когда использовать

Когда в `posts/inbox/` появился новый пост от агентов.

## Что делаю

1. Читаю `posts/inbox/post-NN.md` — текст поста
2. Смотрю `posts/inbox/post-NN.png` — дизайн (открой файл)
3. Проверяю по чеклисту из `global/standards.md`
4. Если ок — перемещаю в `posts/approved/`
5. Если нужны правки — правлю текст прямо в файле и перемещаю в `posts/approved/`
6. Если серьёзные проблемы — перемещаю в `posts/drafts/` и добавляю `## Замечания` внизу файла

## Как перемещать файлы

```bash
# Находясь в папке проекта клиента
# Одобрить пост:
mv posts/inbox/post-01.md posts/approved/
mv posts/inbox/post-01.html posts/approved/
mv posts/inbox/post-01.png posts/approved/

# Вернуть на доработку:
mv posts/inbox/post-01.md posts/drafts/
mv posts/inbox/post-01.html posts/drafts/
mv posts/inbox/post-01.png posts/drafts/
```

Или скажи Claude: «Одобри post-01» / «Верни post-01 на доработку»

## Что проверять в тексте

- Нет опечаток и грамматических ошибок
- Тон соответствует клиенту (читай context.md если не уверена)
- CTA есть и он конкретный (не «пишите нам», а конкретное действие)
- Пост не слишком длинный / короткий для платформы
- Нет запрещённых слов из context.md клиента

## Что проверять в дизайне (PNG)

- Текст читается, хороший контраст
- Цвета соответствуют бренду клиента
- Ничего не обрезано по краям
- Есть название/логотип бренда
