# Скилл: Применить фидбек заказчика

## Когда использовать

Заказчик прислал правки на пост — нужно зафиксировать и применить.

## Шаг 1: Записать фидбек

Создай файл `feedback/fb-01.md` (следующий номер):

```markdown
# Фидбек 01
Дата: [дата получения]
Пост: post-NN
Источник: заказчик / сама заметила

## Что сказал заказчик

[Дословно или точный пересказ]

## Что нужно изменить

- [ ] [конкретное изменение]
- [ ] [конкретное изменение]

## Статус
- [ ] Передано агенту
- [ ] Исправлено
- [ ] Одобрено заказчиком
```

## Шаг 2: Простые правки — делаю сама

Если правка простая (поменять слово, убрать хэштег, сократить абзац, изменить CTA):
- Открой `posts/approved/post-NN.md`
- Внеси правку прямо в файл
- Отметь в feedback: `- [x] Исправлено`

## Шаг 3: Сложные правки — передаю агенту

Если нужно переписать пост или переделать дизайн:
- Скажи Claude: «Примени фидбек из feedback/fb-01.md к post-NN»
- Claude запустит агента с правками
- После исправления отметь: `- [x] Передано агенту`, `- [x] Исправлено`

## Шаг 4: Новый дизайн

Если изменился текст и нужен новый PNG:
```bash
cd /путь/до/smm-system
node tools/render-html.js projects/{client}/posts/approved/post-NN.html projects/{client}/posts/approved/post-NN.png
```
Или скажи Claude: «Перегенерируй PNG для post-NN»
