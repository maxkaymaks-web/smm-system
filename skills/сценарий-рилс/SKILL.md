# /сценарий-рилс

Создаёт полные сценарии Instagram Reels строго по паттернам реальных роликов конкурентов.

---

## Что читать перед началом (обязательно)

```
projects/{ProjectID}/context.md                — клиент, ЦА, tone of voice, соцсети клиента
projects/{ProjectID}/analytics/competitors.md  — аккаунты, топ-ролики, паттерны ниши
projects/{ProjectID}/strategy.md               — стратегия (если заполнена)
```

Если ProjectID не указан — спросить.

## Шаг 1 — Анализ реальных роликов конкурентов (fal.ai)

Перед написанием сценария — смотреть на реальные видео, а не только на цифры.

### Выбор роликов для анализа

Из `analytics/competitors.md` взять топ-ролики (выше медианы каждого аккаунта). Приоритет:
1. Ролики с наибольшим числом просмотров
2. Ролики похожего формата на планируемый сценарий
3. 3–5 роликов = достаточно для одного сценария

### Анализ видео через fal.ai

```js
import { fal } from "@fal-ai/client";

const imgUrl = await fal.storage.upload(new Blob([frameData], { type: "image/jpeg" }));

const result = await fal.run("fal-ai/any-llm/vision", {
  input: {
    model: "google/gemini-2.5-flash",
    image_url: imgUrl,
    prompt: `Ты анализируешь кадр из Instagram Reels для фитнес-ниши.
    Опиши подробно:
    1. ЧТО в кадре: субъект, конкретное действие, ракурс съёмки (сбоку/спереди/сверху/POV)
    2. МОНТАЖ: быстрая нарезка / статика / slow-mo / freeze frame / эффекты
    3. ТЕКСТ на экране: точный текст, позиция (верх/низ/центр), размер (крупный/мелкий), стиль
    4. ОЗВУЧКА: есть ли голос за кадром или субтитры (пометить отдельно от текст-оверлея)
    5. НАСТРОЕНИЕ: эмоция, атмосфера, цветокоррекция
    6. ХУК: что именно делается в первые 2 секунды
    7. CTA внутри видео: есть ли призыв лайкнуть / сохранить / отправить / прокомментировать
    Ответ в JSON.`
  }
});
```

**Важно: сценарий пишется строго на основе того, что реально увидели в роликах-аналогах. Не придумывать структуру — копировать паттерн и адаптировать.**

### Определение музыки

**Способ 1 — fal.ai audio analysis:**
```js
const audioUrl = await fal.storage.upload(audioBlob);
const result = await fal.run("fal-ai/any-llm", {
  input: {
    model: "google/gemini-2.5-flash",
    prompt: `Определи трек из этого аудио:
    1. Название и исполнитель (если распознаётся)
    2. Жанр и BPM
    3. Почему работает для Reels
    Audio URL: ${audioUrl}`
  }
});
```

**Способ 2 — поиск трендов (всегда искать конкретную ссылку):**
- **Instagram**: открыть ролик-аналог → тапнуть трек → копировать ссылку на трек (`instagram.com/reels/audio/...`)
- **Spotify**: плейлисты «Viral Hits», «Trending Now», «Reels Hits» → ссылка на трек (`open.spotify.com/track/...`)
- **WebSearch**: `"trending reels music [месяц год] fitness"` — найти конкретные названия + ссылки

**Правило: «Original Audio» в Reels ≠ нет музыки.** Автор мог не тегнуть трек. Определять на слух или через распознавание.

**В сценарий всегда вставлять прямую ссылку на трек**, а не только название.

## Шаг 2 — Выбор формата

Каждый сценарий — один из 5 типов. Чередовать в контент-плане:

| Тип | Паттерн | Потенциал | Сложность съёмки |
|-----|---------|-----------|-----------------|
| **Образовательный** | «5 ошибок», «лайфхак», «почему не работает» | 1M+ | низкая–средняя |
| **Relatable** | Узнаваемая ситуация, юмор, «я и зал» | 100k+ | низкая |
| **Эстетика** | Атмосфера, тело в движении, настроение, без слов | вирус непредсказуем | средняя |
| **Вирусный** | Мем, «отправь другу», тренд-формат, фраза-триггер | 500k–3M+ | низкая |
| **Мотивация** | Трансформация, личная история, до/после | переменный | низкая |

**Правило 80/20:**
- 80% — нарезки (монтаж готового материала, текст поверх видео, мемы)
- 20% — разговорники (голос за кадром, говорящая голова)

## Шаг 3 — Хук (обязательный элемент каждого рилса)

Хук — первые 2 секунды. Без хука нет досмотра = нет дистрибуции.

### Типы хуков (экспериментировать поочерёдно)

| Тип хука | Пример | Когда работает |
|----------|--------|----------------|
| **Боль-вопрос** | «Почему не растут ягодицы?» | Образовательный |
| **Провокация-факт** | «Ты делаешь это неправильно» | Образовательный, вирусный |
| **Число** | «3 причины / 5 ошибок / 1 лайфхак» | Образовательный |
| **Узнай себя** | «Это ты в зале по понедельникам» | Relatable |
| **Триггер-фраза** | «Тот, кто знает разницу между болью и усталостью» | Мотивация, вирусный |
| **Обещание** | «Один кадр который изменил мою тренировку» | Эстетика, трансформация |
| **Отправь другу** | «Отправь подруге, которая «начнёт с понедельника»» | Вирусный |

**Хук работает без звука.** Текст на экране = обязателен в первые 2 секунды.

## Шаг 4 — Музыка (трендовая, всегда с конкретной ссылкой)

Для каждого сценария — конкретный трек + прямая ссылка.

| Тип | Настроение | Примеры жанров |
|-----|-----------|----------------|
| Образовательный | Нейтральный фон | Lo-fi, ambient, chill phonk |
| Relatable/юмор | Ироничный, лёгкий | Поп, viral TikTok clips |
| Эстетика | Атмосферный, кино | Неоклассика, Izzamuzzic |
| Вирусный | Энергичный, узнаваемый | Phonk, drill, вирусный поп |
| Мотивация | Нарастающий | Эпик, indie, original audio |

**Как найти ссылку:**
1. Instagram: тапнуть трек в ролике-аналоге → скопировать URL аудио
2. Spotify: найти трек → Share → Copy Link
3. WebSearch с запросом на конкретный трек → первая ссылка на Spotify/YouTube

---

## Шаг 5 — CTA внутри видео (обязателен)

CTA внутри видео напрямую влияет на статистику (лайки, комменты, репосты, сохранения), а это буст дистрибуции.

**Форматы CTA в видео:**
- Финальный кадр с текстом: «Сохрани, чтобы не потерять» → буст сохранений
- «Отправь подруге, которая [узнаваемая ситуация]» → буст репостов
- «Ставь ❤️ если делала так же» → буст лайков
- «Напиши в комменты [вопрос]» → буст комментариев
- «Пиши [слово] в директ» → лиды

Выбирать CTA под тип контента. Образовательный → «Сохрани». Relatable → «Отправь». Вирусный → «❤️ если узнала себя».

---

## Шаг 6 — Описание (подпись) — экспериментировать

Ротировать форматы, не использовать один постоянно:

| Формат подписи | Когда | Пример |
|----------------|-------|--------|
| **TG-тег** | Образовательный, продающий | «Полная программа — @[username из context.md]» |
| **Подробный текст** | Образовательный | Развёрнутое описание темы, 200–400 симв. |
| **Короткий комментарий** | Relatable, эстетика | 1–2 строки, tone of voice |
| **Только хэштеги** | Тест | 3–5 нишевых хэштегов |
| **Вопрос к аудитории** | Вовлечение | «А ты делаешь так же?» |
| **Пустая подпись** | Тест виральности | — |

**TG всегда через @username из `context.md` — не «ссылка в шапке».**

## Шаг 7 — Адаптация под проект

Берём паттерн из чужого рилса, адаптируем:
1. **Tone of voice** — звучит как клиент, не как копия
2. **Контекст** — тема актуальна для ЦА клиента
3. **Продукт** — CTA ведёт на нужный канал/продукт клиента
4. **TG username** — конкретный @username из `context.md`

## Выходной документ — HTML (дизайн-система bit&pix)

Создавать `scenario.html` в той же дизайн-системе что контент-план.

### Логотип — всегда GitHub raw URL (работает в любом браузере)
```html
<img src="https://raw.githubusercontent.com/maxkaymaks-web/smm-system/main/global/brand/logo.png" alt="bit&pix агент" style="height:48px">
```

Локальные относительные пути (`../`) не работают при открытии через `file://` в браузере.

### Шрифт
```css
@font-face {
  font-family: 'Nata Sans';
  src: url('../../../../../global/brand/NataSans-VariableFont_wght.ttf') format('truetype');
  font-weight: 100 900;
}
```

### CSS-переменные
```css
--bg: #FEFEFF; --bg-alt: #F6FAFC; --text: #0B0B0B; --accent: #FF1E1E;
--border: rgba(11,11,11,0.09); --muted: rgba(11,11,11,0.42);
--r: 10px; --r-sm: 5px; --r-pill: 20px;
--font: 'Nata Sans', -apple-system, sans-serif;
```

### Структура HTML-документа

```
HEADER — лейбл + заголовок (тема рилса) + подзаголовок (дата, тип, аналог) + лого

BADGES ROW — тип формата · хронометраж · аналог со ссылкой

БЛОК «ХУК» — красная рамка, крупный текст хука, тип хука

РАСКАДРОВКА — нумерованные кадры:
  каждый: номер + тайминг + ЧТО СНЯТЬ (конкретные действия из анализа) +
  ТЕКСТ НА ЭКРАНЕ (что показывается) + ОЗВУЧКА (что говорить, если есть) + монтаж

ИНСТРУКЦИИ — сетка 2×N (ракурс, камера, свет, одежда, монтаж, CTA в видео)

МУЗЫКА — трек + прямая ссылка + обоснование + альтернатива

ПОДПИСЬ + ХЭШТЕГИ — два блока рядом, с @username для TG

ПУБЛИКАЦИЯ — время, обложка, геометка

FOOTER — {Client} · Сценарий #{N} · дата | bit&pix агент
```

**Блоки «аналог vs адаптация» и «чеклист» — не создавать.**
