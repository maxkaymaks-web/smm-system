## Шаг 1 — Анализ реальных роликов конкурентов (fal.ai)

Перед написанием сценария — смотреть на реальные видео, а не только на цифры.

### Выбор роликов для анализа

Из `analytics/competitors.md` взять топ-ролики (выше медианы каждого аккаунта). Приоритет:
1. Ролики с наибольшим числом просмотров в нише
2. Ролики похожего формата на планируемый сценарий
3. 3–5 роликов = достаточно для одного сценария

### Анализ видео через fal.ai

```js
// 1. Скачать превью/кадры ролика
// URL берём из competitors.md или Apify-данных

// 2. Загрузить в fal.ai storage
import { fal } from "@fal-ai/client";
const imgUrl = await fal.storage.upload(new Blob([frameData], { type: "image/jpeg" }));

// 3. Анализировать через vision
const result = await fal.run("fal-ai/any-llm/vision", {
  input: {
    model: "google/gemini-2.5-flash",
    image_url: imgUrl,
    prompt: `Ты анализируешь кадр из Instagram Reels для фитнес-ниши.
    Опиши подробно:
    1. ЧТО в кадре: субъект, действие, окружение, угол съёмки
    2. МОНТАЖ: быстрая нарезка / статика / slow-mo / эффекты
    3. ТЕКСТ на экране: точный текст, позиция, стиль (крупный/мелкий/субтитры)
    4. НАСТРОЕНИЕ: эмоция, атмосфера, цветокоррекция
    5. ХУК: что цепляет в первые 2 секунды по этому кадру
    6. ПАТТЕРН: к какому формату относится (5 ошибок / relatable / эстетика / лайфхак / трансформация / мотивация)
    Ответ в JSON.`
  }
});
```

### Определение музыки

**Способ 1 — fal.ai audio analysis (если есть аудио дорожка):**
```js
// Загрузить аудио-файл
const audioUrl = await fal.storage.upload(audioBlob);
const result = await fal.run("fal-ai/any-llm", {
  input: {
    model: "google/gemini-2.5-flash",
    prompt: `Это аудио из Instagram Reels. Определи:
    1. Название трека и исполнитель (если распознаётся)
    2. Жанр и настроение
    3. Темп (BPM примерно)
    4. Почему этот трек работает для Reels (вирусность / настроение / ритм)
    Audio URL: ${audioUrl}`
  }
});
```

**Способ 2 — поиск трендовых треков:**
- Проверить `music.instagram.com` (trending audio section)
- Spotify: плейлисты «Viral Hits», «Trending Now», «Reels Hits»
- WebSearch: «trending reels music [месяц год]» / «trending audio instagram [ниша]»
- TikTok creative center: `ads.tiktok.com/business/creativecenter/music`

**Правило:** «Original Audio» в Reels ≠ нет музыки. Это значит что автор не тегнул трек. Сам ролик может использовать известный трек — анализировать на слух или через распознавание.

## Шаг 2 — Выбор формата

Каждый сценарий — один из 5 типов. Чередовать в контент-плане:

| Тип | Паттерн | Потенциал | Сложность съёмки |
|-----|---------|-----------|-----------------|
| **Образовательный** | «5 ошибок», «лайфхак», «почему не работает» | 1M+ | низкая–средняя |
| **Relatable** | Узнаваемая ситуация, юмор, «я и зал» | 100k+ | низкая |
| **Эстетика** | Атмосфера, тело в движении, настроение, без слов | вирус непредсказуем | средняя |
| **Вирусный** | Мем, «отправь другу», тренд-формат, фраза-триггер | 500k–3M+ | низкая |
| **Мотивация** | Трансформация, личная история, до/после | переменный | низкая |

**Правило 80/20:**
- 80% — нарезки (монтаж готового материала, текст поверх видео, мемы)
- 20% — разговорники (голос за кадром, говорящая голова)

**Правило эксперимента:** каждый месяц пробуем все 5 типов. Смотрим что даёт охват у конкретного клиента, масштабируем.

## Шаг 3 — Хук (обязательный элемент каждого рилса)

Хук — первые 2 секунды. Без хука нет досмотра = нет дистрибуции.

### Типы хуков (экспериментировать поочерёдно)

| Тип хука | Пример | Когда работает |
|----------|--------|----------------|
| **Боль-вопрос** | «Почему не растут ягодицы?» | Образовательный |
| **Провокация-факт** | «Ты делаешь это неправильно» | Образовательный, вирусный |
| **Число** | «3 причины / 5 ошибок / 1 лайфхак» | Образовательный |
| **Узнай себя** | «Это ты в зале по понедельникам» | Relatable |
| **Триггер-фраза** | «Тот, кто знает разницу между болью и усталостью» | Мотивация, вирусный |
| **Обещание** | «Один кадр который изменил мою тренировку» | Эстетика, трансформация |
| **Отправь другу** | «Отправь подруге, которая «начнёт с понедельника»» | Вирусный |

**Правило хука:** работает без звука. Текст на экране = обязателен в первые 2 секунды.

## Шаг 4 — Музыка (трендовая, не generic)

Для каждого сценария — конкретный трек или направление, не просто «добавь музыку».

### По типу контента

| Тип | Музыкальное настроение | Примеры жанров |
|-----|----------------------|----------------|
| Образовательный | Нейтральный, ненавязчивый фон | Lo-fi, ambient, chill phonk |
| Relatable/юмор | Ироничный, лёгкий, узнаваемый | Поп, viral TikTok clips |
| Эстетика | Атмосферный, кино | Неоклассика, инструментал, Izzamuzzic |
| Вирусный | Энергичный, узнаваемый хит | Phonk, drill, вирусный поп |
| Мотивация | Нарастающий, вдохновляющий | Эпик, indie, original audio |

### Как искать конкретный трек

1. **Instagram Reels**: открыть ролик-аналог → тапнуть на трек внизу → посмотреть сколько рилсов с ним (100k+ = трендовый)
2. **Spotify**: плейлисты «Reels Ready», «Phonk», «Viral Russia», «Chill Vibes»
3. **WebSearch**: `"trending reels music may 2026 fitness"` / `"топ музыка для рилс 2026 фитнес"`
4. **fal.ai + Gemini**: загрузить аудио из ролика-аналога → распознать трек

## Шаг 5 — Описание (подпись) — экспериментировать

Не используть один формат для всех. Ротировать:

| Формат подписи | Когда | Пример |
|----------------|-------|--------|
| **CTA в TG** | Образовательный, продающий | «Полная программа — в моём TG (ссылка в шапке)» |
| **Подробное описание темы** | Образовательный с длинным текстом | Развёрнутый текст по теме ролика (200–400 симв.) |
| **Короткий комментарий** | Relatable, эстетика | 1–2 строки, tone of voice клиента |
| **Только хэштеги** | Тест | 3–5 нишевых хэштегов, без текста |
| **Вопрос к аудитории** | Вовлечение | «А ты делаешь так же? Пиши в комментах» |
| **Пустая подпись** | Тест виральности без текста | — |

**Цель:** не угадать заранее, а тестировать каждый формат и смотреть на комментарии + share.

## Шаг 6 — Адаптация под проект

Берём чужие связки — но адаптируем. Обязательно проверить:

1. **Tone of voice** — текст и подпись звучат как клиент, а не как копия оригинала
2. **Контекст** — тема актуальна для ЦА клиента (из `context.md`)
3. **Продукт** — если нужно, CTA ведёт на нужный продукт/канал клиента
4. **Локация** — если зал конкретный, геометки и контекст соответствуют

### Структура HTML-документа (сценарий)

```
HEADER
  ├── тип + номер сценария (лейбл мелкий)
  ├── заголовок — тема рилса
  ├── подзаголовок — дата · тип · аналог
  └── логотип (правый угол)

BADGES ROW
  ├── тип формата (образовательный / relatable / etc)
  ├── хронометраж цели
  └── аналог-аккаунт (со ссылкой)

БЛОК «ХУК»
  акцентный блок с красной рамкой, крупный текст хука + тип хука

РАСКАДРОВКА
  нумерованные кадры (как .shot в сценарии съёмки)
  каждый: номер + время + визуал + монтаж + текст + аудио

ИНСТРУКЦИИ
  сетка 2×N (.how-grid / .how-cell)

МУЗЫКА
  блок с треком, жанром, ссылкой, обоснованием

ПОДПИСЬ + ХЭШТЕГИ
  два блока рядом: текст подписи + хэштеги

ПУБЛИКАЦИЯ + ЧЕКЛИСТ
  настройки и чеклист перед публикацией

FOOTER
  left: {Client} · Сценарий #{N} · ДД.ММ.ГГГГ
  right: bit&pix агент
```
