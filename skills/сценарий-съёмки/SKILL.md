# Скилл: СЦЕНАРИЙ СЪЁМКИ

## Когда вызывать

Пользователь просит:
- «сделай сценарий съёмки»
- «напиши инструкции для заказчика что снять»
- «ТЗ на съёмку»
- «инструкции как снимать»
- «что снять для поста»

---

## Что собрать перед созданием

1. **Проект** — какой клиент / папка проекта в smm-system
2. **Дата публикации** — когда выйдет пост
3. **Дедлайн съёмки** — когда нужны материалы
4. **Формат поста** — фото-карусель / видео / reels / клип
5. **Контекст** — что за событие, продукт, момент; что клиент хочет показать
6. **Люди** — кто будет снимать, кто в кадре

Если чего-то не хватает — спроси до начала работы.

## Структура HTML-документа

### Заголовок (header)
- Лейбл: `СЦЕНАРИЙ СЪЁМКИ · [НАЗВАНИЕ КЛИЕНТА]` (uppercase, muted)
- Заголовок: название поста в кавычках — ёмкое, образное
- Подзаголовок: `Публикация [дата] · [Платформа] · [тип контента]`
- Логотип bit&pix агент (правый верхний угол)

### Бейджи (badges row)
- **Формат** — «Фото, 5–8 кадров» или «Видео 30–60 сек»
- **Дедлайн съёмки** — дата, акцентный (красный) бейдж

### Блок 1: «ЧТО СНИМАТЬ — ПОЭПИЗОДНО»
Нумерованный список кадров/эпизодов. На каждый:
- Заголовок кадра (жирный, 14px)
- Описание: что именно снять, как выглядит, что важно поймать (2–3 предложения)

**Принципы для кадров:**
- Порядок: подготовка → процесс → ключевой момент → детали → финал/команда
- Минимум 5, максимум 8 кадров для фото; 4–7 эпизодов для видео
- Каждый кадр решает конкретную задачу — «доказательность», атмосфера, эмоция, экспертиза
- Конкретика: не «снимите процесс», а «руки включают аппарат, виден экран с индикатором»

### Блок 2: «КАК СНИМАТЬ»
Сетка 2×N из ячеек. Обязательные параметры:

| Параметр | Для фото | Для видео |
|----------|----------|-----------|
| Ориентация | Горизонт. (16:9) или квадрат (1:1) | Вертикаль (9:16) или горизонт. (16:9) |
| Количество исходников | 15–20 кадров, отберём лучшие | — |
| Длительность | — | Общая + каждый эпизод |
| Свет | Натуральный / зальный / студийный | То же |
| Движение камеры | Статика, держать двумя руками | Медленно, без тряски |
| Люди в кадре | Кто обязательно, кто опционально | То же |

Если параметр занимает всю строку — `class="full"`.

### Блок 3: «СОВЕТЫ ПЕРЕД СЪЁМКОЙ»
3–5 советов в красных плашках. Формат: **Жирный ключевой тезис** — развёртка одним предложением.

Типичные советы:
- Снимать больше чем нужно, прислать всё сырое
- Убрать лишнее из кадра (личные вещи, беспорядок)
- Свет — главное условие качества
- Для мероприятий: снимать в перерывах тоже, не только доклад
- Для видео: держать телефон горизонтально / положить на поверхность

### Footer
- Левая: `[Клиент] · Пост [дата] · сценарий v1`
- Правая: точка-акцент + `bit&pix агент`

## CSS-компоненты

### .shot — кадр/эпизод
```css
display: flex; gap: 16px; align-items: flex-start;
padding: 15px 0; border-bottom: 1px solid var(--border);
```
- `.shot-num` — чёрный круг 28px, белый номер
- `.shot-title` — 14px, font-weight 700
- `.shot-desc` — 12.5px, color var(--muted), line-height 1.55

### .how-grid — сетка параметров
```css
display: grid; grid-template-columns: 1fr 1fr;
gap: 1px; background: var(--border); border-radius: 8px; overflow: hidden;
```
- `.how-cell` — белый фон, padding 15px 18px
- `.how-cell.full` — `grid-column: 1 / -1`
- `.how-label` — 9.5px, uppercase, letter-spacing 2px, muted
- `.how-value` — 13.5px, font-weight 500

### .tip — совет
```css
background: rgba(255,30,30,0.05);
border: 1px solid rgba(255,30,30,0.15);
border-radius: var(--r-sm); padding: 12px 16px;
```
- `::before { content: '✦'; color: var(--accent); margin-right: 8px; }`
- `strong` — color var(--accent)

### .badge — бейдж
```css
display: inline-flex; align-items: center; gap: 8px;
padding: 7px 14px; border-radius: var(--r-pill);
border: 1.5px solid var(--border); background: var(--bg-alt);
```
- `.badge.accent` — красный фон и рамка, цвет `#C00`
- `.badge-label` — 9px, uppercase, letter-spacing 2px, muted

## Пример структуры для фото-поста (мероприятие)

**Кадры:**
1. До начала — ноутбук/материалы, атмосфера «готовимся»
2. Зал — общий вид, баннер с названием события
3. Выступление — спикер сбоку, зал на фоне
4. Ключевой слайд — крупно, чётко
5. Дискуссия — живой разговор
6. Командное фото — неформально
7. Деталь — бейдж / программа / буклет

**Пример структуры для видео (студия / продукт):**
1. Открытие / вход в пространство
2. Общий вид — чистота и порядок
3. Ключевое действие (руки, процесс)
4. Расходники / детали / оборудование
5. Атмосферная деталь
6. Финал — готовность / результат
