# /ежедневный-брифинг — Ежедневный брифинг по контент-планам

Запускается каждое утро. Сканирует все проекты, фильтрует по срокам, выдаёт список задач на день.

---

## Алгоритм выполнения

### 1. Найти все проекты

```
ls /Users/maxkaymaks/CLAUDE/smm-system/projects/
```

Для каждого проекта читать `content-plan.md`.

### 2. Распарсить таблицу постов

Из каждой строки таблицы извлечь:
- `#` — номер поста
- `Дата` — дата публикации (формат DD.MM)
- `Рубрика` / `Тема`
- `Статус` — текущий статус

Год берём текущий (из системной даты). Если дата уже прошла в этом году — считать следующий год.

### 3. Фильтры

**Приоритет 1 — 🔥 Горит (нам делать)**
Условие: статус НЕ `опубликовано` И НЕ `на согласовании` И дней до публикации ≤ 3.
Если срок уже прошёл — пометить `(просрочен)`.

**Приоритет 2 — 📋 Скоро (нам делать)**
Условие: статус = `draft` или `черновик`, дней до публикации 4–7.

**Приоритет 3 — 📦 Запросить материалы у заказчика**
Условие: статус = `ждём материалы`, дней до публикации ≤ 7.

**Приоритет 4 — ⏳ На согласовании**
Условие: статус содержит `на согласовании`.

**Статистика — просто считаем:**
- Всего постов в плане
- Опубликовано
- В работе (draft)
- Ждём материалы
- На согласовании

### 4. Формат вывода

```
📅 Брифинг [DD месяц YYYY]
────────────────────────

🔥 ГОРИТ (нужно сделать сегодня/срочно)
• [Проект] Пост #NN — [Тема] — публикация DD.MM (через N дн / ПРОСРОЧЕН)

📋 В РАБОТЕ (дедлайн через 4–7 дней)
• [Проект] Пост #NN — [Тема] — публикация DD.MM (через N дн)

📦 ЗАПРОСИТЬ МАТЕРИАЛЫ У ЗАКАЗЧИКА
• [Проект] Пост #NN — [Тема] — публикация DD.MM (через N дн)

⏳ НА СОГЛАСОВАНИИ
• [Проект] Пост #NN — [Тема] — отправлен заказчику

────────────────────────
📊 Статистика
[Проект 1]: X постов | опубликовано N | draft N | ждём материалы N | согласование N
[Проект 2]: ...
```

Если в разделе нет пунктов — раздел не показывать.

### 5. Правила подсчёта дней

```python
from datetime import date, timedelta
today = date.today()
pub_date = date(today.year, month, day)
if pub_date < today:
    pub_date = date(today.year + 1, month, day)
days_left = (pub_date - today).days
```

### 6. Маппинг статусов

| Строка в файле | Категория |
|----------------|-----------|
| `опубликовано` | published |
| `на согласовании` | review |
| `ждём материалы` | waiting |
| `draft`, `черновик`, `draft · согласовано` | draft |

Статус `draft · согласовано` = контент согласован, но ещё не опубликован → считать как `draft`.
